# 数字惠农系统 - 前端 UI/UX 设计文档

## 1. 文档概述

### 1.1 文档目的
本文档旨在为"数字惠农APP"及配套"惠农OA后台管理系统"的前端开发提供统一的UI（用户界面）和UX（用户体验）设计指导，确保产品最终呈现给用户时具有一致性、易用性和良好的用户体验。

### 1.2 参考文档
- 《需求说明文档 (PRD) - 数字惠农APP及OA后台管理系统》 (`agent/demand/PRD-2025-5-29.md`)
- 《用户故事地图 (User Story Map) - 数字惠农APP及OA后台管理系统》 (`agent/demand/User_Story_Map.md`)

## 2. 设计原则

遵循以下设计原则，以确保产品的质量和用户满意度：

-   **一致性**: 整体操作流程、界面元素、文案风格保持一致。
-   **反馈性**: 用户每一步操作都应有及时、明确的反馈。
-   **容错性**: 提供撤销、修改等功能，对于重要操作前进行确认提醒。防止用户误操作。
-   **易学性**: 界面简洁直观，减少用户学习成本。对于复杂功能提供引导。
-   **高效性**: 优化操作路径，减少不必要的步骤，提高用户完成任务的效率。
-   **可访问性**: 兼顾不同用户群体的需求，确保界面信息清晰易懂，操作便捷。

## 3. 目标用户

本产品主要面向以下两类用户群体：

### 3.1 数字惠农APP用户
-   **个体农户 (如 李明)**: 年龄稍大，对智能手机使用有一定基础，但可能不熟悉复杂操作。关注实用性、易用性，对小额贷款、农机租赁、政策资讯有实际需求。
-   **家庭农场主/合作社负责人 (如 王芳)**: 年龄相对年轻，熟练使用各类App。对融资、农场管理、市场信息有较高要求，注重效率和专业性。

### 3.2 惠农OA后台管理系统用户
-   **基层信贷审批员/农机租赁管理员 (如 张伟)**: 熟悉办公软件，日常处理大量申请，追求高效、准确的系统操作。
-   **风控部门主管/运营经理 (如 刘经理)**: 关注数据分析、风险控制和流程优化，对系统稳定性、灵活性有较高要求。

## 4. 整体布局与风格

### 4.1 数字惠农APP

-   **整体风格**: 亲切、友好、可靠。设计应简洁明了，符合农村用户审美和使用习惯。
-   **色彩**: 主色调可选用代表农业、丰收、信任的颜色，如绿色、蓝色、橙色系的暖色调。辅助色用于强调和区分。
-   **字体**: 选择清晰易读的无衬线字体，正文字号可适当偏大（例如14-16px），重要信息标题字号（18-22px）。
-   **图标**: 图标设计应表意清晰，易于理解，风格统一。
-   **布局**:
    -   **首页**: 突出核心服务入口（贷款、租赁、资讯、惠农服务等），可采用宫格或列表布局。顶部可设置Banner用于推广重要信息或活动。
    -   **导航**: 底部Tab导航栏，用于切换主要功能模块。
    -   **列表页**: 信息展示清晰，条目间距适中，关键信息突出。支持下拉刷新、上拉加载更多。
    -   **详情页**: 信息结构化展示，重点内容醒目。操作按钮位置方便。

### 4.2 惠农OA后台管理系统

-   **整体风格**: 专业、高效、严谨。注重信息层级和操作便捷性。
-   **色彩**: 主色调可选用蓝色或灰色系，体现专业和稳定。辅助色用于高亮、警告等状态。
-   **字体**: 选择清晰的无衬线字体，如微软雅黑、PingFang SC等，标准字号（12-14px）。
-   **图标**: 采用通用的后台管理系统图标，保持一致性。
-   **布局**:
    -   **经典布局**: 左侧为导航菜单（可收起），右侧为内容显示区域。顶部为通用操作区（如用户信息、消息通知、退出登录）。
    -   **导航**: 树形导航菜单，层级清晰。
    -   **内容区**: 多采用表格、表单、卡片等形式展示数据和进行操作。

## 5. 关键页面原型参考/线框图说明

以下是对核心页面/模块的初步设计构想，详细设计将在具体迭代中进行。

### 5.1 数字惠农APP

-   **注册/登录页**:
    -   提供手机号+验证码登录/注册方式。
    -   界面简洁，操作指引清晰。
    -   包含隐私政策和服务协议链接。
-   **首页**:
    -   顶部Banner区。
    -   核心功能入口（贷款、租赁、资讯、我的）采用大图标+文字宫格布局。
    -   下方可有"最新政策"、"热门资讯"等信息流推荐。
-   **贷款模块**:
    -   **产品列表页**: 卡片式展示各贷款产品，包含产品名称、主要特点（如最高额度、利率范围）、申请按钮。
    -   **产品详情页**: 详细介绍产品信息、申请条件、所需材料、利率、期限等。
    -   **申请表单页**: 分步式表单，每步信息量适中。输入框标签清晰，提供示例或帮助。支持OCR识别身份证、银行卡。材料上传界面支持拍照、选择相册，可预览已上传文件。
    -   **我的贷款/申请进度页**: 列表展示申请记录，状态步骤条（如审核中、已通过、已放款）清晰展示当前进展。
-   **农机租赁模块**:
    -   **农机列表页**: 支持按类型、区域筛选。地图模式和列表模式切换。卡片展示农机图片、名称、类型、价格、距离等。
    -   **农机详情页**: 展示农机高清图片、详细参数、租赁价格、用户评价、农机主信息（部分隐藏，确认后可见）。
    -   **租赁订单提交页**: 选择租赁时间、地点，确认费用，提交订单。
    -   **我的租赁订单页**: 列表展示租赁订单，状态清晰（待确认、待支付、待使用等）。
-   **农业资讯模块**:
    -   **资讯列表页**: 卡片式或列表式展示资讯，包含标题、摘要、来源、发布时间、缩略图。支持分类切换。
    -   **资讯详情页**: 图文混排，阅读体验良好。提供分享、收藏功能。
-   **用户中心 (我的)**:
    -   显示用户头像、昵称。
    -   入口：我的申请、我的订单、我的资料、实名认证、消息通知、设置等。

### 5.2 惠农OA后台管理系统

-   **登录页**:
    -   用户名、密码输入框。
    -   验证码（可选）。
    -   登录按钮。
-   **主工作台/首页**:
    -   展示核心业务数据概览（如待处理申请数、用户总数等）。
    -   常用功能快捷入口。
    -   系统公告或通知。
-   **贷款审批模块**:
    -   **申请列表页**: 表格展示贷款申请，包含申请编号、申请人、产品名称、申请金额、申请时间、当前状态等。支持筛选、排序、分页。操作列包含"查看详情"、"审批"等按钮。
    -   **审批详情页**: 分区域展示申请人信息、申请信息、AI审批建议（风险点、评分等）、历史审批记录。审批操作区（通过、拒绝、补充材料等按钮），审批意见填写框。
-   **农机租赁管理模块**:
    -   **租赁请求列表页**: 表格展示租赁请求，包含订单号、用户、农机名称、租赁时间、状态等。
    -   **租赁详情/处理页**: 展示订单详情，AI辅助建议，人工处理操作（确认订单、指派农机、取消订单等）。
-   **用户管理模块**:
    -   **用户列表页**: 表格展示APP用户信息，支持搜索、筛选。可查看用户详情，进行账户状态管理。
-   **内容管理模块**:
    -   列表展示已发布内容（资讯、政策等）。
    -   提供新建、编辑、删除内容的表单和编辑器。

## 6. 组件库与设计规范 (初步)

前端开发将基于 Ant Design (用于OA后台) 和 Ant Design Mobile / React Native Elements (或自定义组件，用于APP) 作为基础组件库，并根据产品风格进行定制。

-   **颜色**:
    -   主色 (Primary): APP (如 #4CAF50 绿色), OA (如 #1890FF 蓝色)
    -   辅助色 (Accent/Secondary): 用于按钮、高亮等。
    -   成功色 (Success): 如 #52C41A
    -   警告色 (Warning): 如 #FAAD14
    -   错误色 (Error): 如 #FF4D4F
    -   文字颜色: 主文字 (#333333), 次要文字 (#666666), 提示文字 (#999999)
    -   背景色: 页面背景、卡片背景等。
-   **字体**:
    -   APP: PingFang SC / San Francisco (iOS), Roboto (Android)。字号：12px, 14px, 16px, 18px, 20px。
    -   OA: Microsoft YaHei, PingFang SC。字号：12px, 14px, 16px。
-   **图标**: 使用统一风格的SVG图标库或字体图标。
-   **按钮**:
    -   类型：主按钮、次按钮、文字按钮、危险按钮。
    -   状态：正常、悬浮(Hover)、点击(Active)、禁用(Disabled)。
    -   尺寸：大、中、小。
-   **表单**:
    -   输入框、选择器、日期选择器、开关、单选/复选框等，样式统一。
    -   清晰的标签和提示信息，校验状态的视觉反馈。
-   **表格 (OA)**:
    -   统一样式，支持分页、排序、筛选。操作列按钮样式。
-   **消息提示**:
    -   全局提示 (Message/Toast): 用于轻量级操作反馈。
    -   对话框 (Modal/Dialog): 用于重要操作确认、信息展示。
    -   通知提醒框 (Notification): 用于系统级通知。
-   **卡片 (Card)**: 用于APP中信息聚合展示。
-   **步骤条 (Steps)**: 用于展示多步骤流程，如贷款申请。
-   **时间轴 (Timeline)**: 用于展示进度或历史记录。

## 7. 交互规范

-   **点击反馈**: 所有可点击元素在点击时应有视觉反馈。
-   **加载状态**: 异步操作（如API请求）时，应显示加载指示器（如Spinner、骨架屏），避免界面无响应。
-   **空状态**: 列表、搜索结果为空时，应有友好的空状态提示，并可能引导用户进行其他操作。
-   **错误处理**:
    -   表单校验错误应在对应表单项下方即时提示。
    -   API请求错误应根据错误类型给出明确提示（如网络错误、业务逻辑错误等）。
-   **手势操作 (APP)**:
    -   列表支持下拉刷新。
    -   图片支持手势缩放。
    -   轮播图支持左右滑动。
-   **键盘交互 (OA)**:
    -   表单支持Tab键切换焦点。
    -   对话框支持Esc键关闭。
    -   常用操作支持快捷键 (可选)。

## 8. 响应式设计 (OA后台)

OA后台管理系统应支持在主流桌面浏览器上良好显示，并能适应不同屏幕分辨率（最低支持1366x768）。关键布局和组件在不同尺寸下能自动调整。
